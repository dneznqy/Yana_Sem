# Офлайн-запуск при недоступности Maven Central

Если исходная машина не может скачивать зависимости (например, доступ к `https://repo.maven.apache.org` блокируется прокси), подготовьте локальный кэш Maven на любой машине с интернетом и перенесите его в каталог проекта.

## Подготовка кэша на машине с интернетом

```bash
# В корне проекта или в любом каталоге с pom.xml
mvn -Dmaven.repo.local=./maven-cache dependency:go-offline
```

Команда скачает все зависимости, нужные для сборки и запуска приложения, в каталог `./maven-cache`.

## Перенос кэша

1. Скопируйте каталог `maven-cache` на целевую машину рядом с проектом (например, в `/workspace/Yana_Sem/maven-cache`).
2. Убедитесь, что у вас достаточно места на диске: кэш Spring Boot 3 занимает сотни мегабайт.

## Запуск с использованием локального кэша

В проекте есть вспомогательный скрипт, который запускает приложение, используя локальный репозиторий:

```bash
./scripts/run-offline.sh maven-cache
```

Если каталог с кэшем находится в другом месте, укажите путь первым аргументом. Скрипт завершится с ошибкой, если каталог не найден, и подскажет, как его создать.

## Сборка и тесты через локальный кэш

Аналогично можно запускать любые Maven-команды, добавляя параметр `-Dmaven.repo.local=./maven-cache`, например:

```bash
mvn -Dmaven.repo.local=./maven-cache clean test
```

Это позволит собирать и тестировать проект в среде без прямого доступа к интернету.
