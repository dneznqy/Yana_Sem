# Пояснительная записка (структура)

Тема: «Информационно-справочная система интернет-магазина»

## 1. Титульный лист

Оформляется по требованиям вашего факультета (университета).

## 2. Оглавление

Формируется автоматически в текстовом редакторе.

## 3. Введение

- Краткое описание предметной области интернет-магазина.
- Актуальность темы (рост онлайн-торговли, необходимость удобного доступа к информации о товарах).
- Цели и задачи курсовой работы.
- Краткое описание используемых технологий (Java, Spring Boot, БД PostgreSQL и др.).

## 4. Аналитический раздел

- Объект и предмет исследования.
- Описание типовой функциональности интернет-магазинов.
- Обзор существующих решений (крупные интернет-магазины, их функции).
- Обоснование выбора стека технологий (Java + Spring Boot + JPA + PostgreSQL, Thymeleaf).

## 5. Проектный раздел

### 5.1. Постановка задачи

- Требования к системе (на основе методички и технического задания).
- Пользовательские роли: администратор и покупатель.
- Основные сценарии работы пользователей.

### 5.2. Архитектура приложения

- Описание трехзвенной архитектуры: клиентский слой (HTML/Thymeleaf), серверный слой (Spring MVC, сервисы),
  слой данных (JPA-сущности, репозитории, PostgreSQL).
- Диаграмма компонентов и/или архитектурная схема.

### 5.3. Модель данных

- Описание сущностей: Пользователь, Роль, Категория, Товар, Заказ.
- Диаграмма классов или ER-диаграмма.
- Связи между таблицами в БД.

### 5.4. Безопасность

- Описание регистрации и авторизации.
- Роли и разграничение прав доступа.
- Использование Spring Security, шифрование паролей (BCrypt).
- Защита от SQL-инъекций за счет ORM (JPA/Hibernate).

### 5.5. Описание интерфейса

- Основные страницы: вход, регистрация, каталог товаров, управление товарами и категориями,
  работа с заказами, раздел «Об авторе», раздел статистики.
- Скриншоты основных экранов с краткими комментариями.

### 5.6. Алгоритмы и бизнес-логика

- Логика оформления заказа, уменьшение остатка товара.
- Логика подсчета статистики (кол-во пользователей, товаров, заказов, среднее время подтверждения заказа).
- Обработка ошибок и вывод понятных сообщений пользователю.

## 6. Технологический раздел

- Описание настроек проекта (Maven, pom.xml).

- Краткое описание структуры проекта (пакеты config, entity, repository, service, controller, templates).
- Инструкция по сборке и запуску (как в README.md).

## 7. Тестирование

- Описание сценариев проверки основных функций:
  - регистрация и вход;
  - просмотр каталога, поиск и сортировка;
  - создание, редактирование и удаление товара (для администратора);
  - создание заказа и подтверждение заказа;
  - просмотр статистики;
  - обработка ошибок (некорректный ввод, недостаток товара и т.д.).
